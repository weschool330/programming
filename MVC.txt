Step 1: Create the Database Migration, Model, and Seeder
Laravel uses Migrations like version control for your database, Models to interact with that data, and Seeders to populate it with test data. We can generate all three at once using an Artisan command in your terminal.

Run this command in your project root:

Bash
php artisan make:model Movie -ms
What this does: > * Creates the Movie model (app/Models/Movie.php).

-m creates the database migration file.

-s creates a database seeder (database/seeders/MovieSeeder.php).

1.1 The Migration File
Open the newly created migration file located in database/migrations/xxxx_xx_xx_xxxxxx_create_movies_table.php. Update the up method to define the columns requested:

PHP
public function up(): void
{
    Schema::create('movies', function (Blueprint $table) {
        $table->id(); // Auto-incrementing primary key
        $table->string('title');
        $table->string('director');
        $table->integer('release_year');
        $table->decimal('rating', 3, 1); // Allows ratings like 8.5 or 10.0
        $table->timestamps(); // Created_at and updated_at tracking
    });
}
1.2 The Movie Model
Open app/Models/Movie.php. We need to tell Laravel which fields are allowed to be filled safely.

PHP
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class Movie extends Model
{
    use HasFactory;

    // Defines which fields can be mass-assigned
    protected $fillable = ['title', 'director', 'release_year', 'rating'];
}

1.4 Activate and Run everything
Open database/seeders/DatabaseSeeder.php and make sure your MovieSeeder runs when you seed the database:

PHP
public function run(): void
{

Now, ensure your .env file is properly connected to your local database (MySQL/SQLite), then run this command to build the tables and insert the data:

Bash
php artisan migrate --seed
Step 2: Controller & Routes
The Controller acts as the brain, fetching data from the Model and passing it to the Views.

2.1 Create the Controller
Run the following command to generate a resource controller:

Bash
php artisan make:controller MovieController --resource
Open app/Http/Controllers/MovieController.php. We only need to implement the index (all movies) and show (single movie) methods for this task. Keep it clean and explicit:

PHP
<?php

namespace App\Http/Controllers;

use App\Models\Movie;
use Illuminate\Http\Request;

class MovieController extends Controller
{
    // 1. Movie Catalog / Main Listing
    public function index()
    {
        $movies = Movie::all(); // Fetches all movies from database
        return view('movies.index', compact('movies')); // Passes data to views/movies/index.blade.php
    }

    // 2. Detailed Movie Page
    public function show(string $id)
    {
        $movie = Movie::findOrFail($id); // Finds movie or returns a 404 error if not found
        return view('movies.show', compact('movie'));
    }
}
2.2 Set up the Routes
Open routes/web.php. We will point our URLs directly to our controller methods.

PHP
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\MovieController;

// Redirect homepage to our movie list
Route::get('/', function () {
    return redirect()->route('movies.index');
});

// Explicit, clean routes for index and show views
Route::get('/movies', [MovieController::class, 'index'])->name('movies.index');
Route::get('/movies/{id}', [MovieController::class, 'show'])->name('movies.show');
